HYPER84

2-D Hyperelastic Solid

MP ME ST <> <> <> <> <> <> PP ED

HYPER84 Element Description

HYPER84 is used for 2-D modeling of solid hyperelastic structures. This element is applicable to rubber-like materials with arbitrarily large displacements and strains. Both compressible and nearly incompressible materials may be modeled. The element can be used either as a plane element (plane strain) or as an axisymmetric ring element. The element is defined either by 4 or 8 nodes. The plane strain option has two degrees of freedom at each node: translation in the nodal x and y directions. The axisymmetric option has an additional degree of freedom at each node: translation in the z direction allowing torsion in the model. The hyperelastic formulation is nonlinear and requires an iterative solution. Large deflections [NLGEOM] must be active in order to update the geometry each substep.

Other options allow you to select the type of strain energy density function, the element printout locations, and reduced integration. See HYPER84 in the ANSYS, Inc. Theory Reference for more details about this element. Similar elements (without the hyperelastic formulation) are PLANE25, PLANE42, PLANE82 and PLANE83. See also elements HYPER56 and HYPER74 for other hyperelastic formulations.

HYPER84 Input Data

The geometry, node locations, and the coordinate system for this element are shown in Figure 84.1: "HYPER84 Geometry". The element input data include the type of element formulation (KEYOPT(1)), the respective nodes, the strain energy function (KEYOPT(2)), the isotropic material properties, and the constants defining the Mooney-Rivlin strain energy function.

Element loads are described in Node and Element Loads. Pressures may be input as surface loads on the element faces as shown by the circled numbers on Figure 84.1: "HYPER84 Geometry". Positive pressures act into the element. Temperatures may be input at the element nodes (for material property evaluation only). The node I temperature T(I) defaults to TUNIF. If all other temperatures are unspecified, they default to T(I). If all corner node temperatures are specified, each midside node temperature defaults to the average temperature of its adjacent corner nodes. For any other input temperature pattern, unspecified temperatures default to TUNIF.

The nodal forces, if any, should be input per unit of depth for a plane strain analysis and on a full 360° basis for an axisymmetric analysis.

The type of element formulation is specified with KEYOPT(1). Both formulations allow a triangular-shape by defining the same node number for nodes K, L, and O (if appropriate). The 4 node option requires less computation time and storage and is recommended for general use. The element coordinate system orientation is as described in Coordinate Systems.

KEYOPT(2) allows a choice of strain energy functions. The Mooney-Rivlin function is valid for a wide range of nearly incompressible rubber and rubber-like materials. It is characterized by Poisson's ratio and constants a10 and a01 (entered with TBDATA or *MOONEY commands), which may be a function of temperature. See Data Tables - Implicit Analysis for the TBDATA command input. The Blatz-Ko function may be applicable to compressible foam-type polyurethane rubber materials. It is a function of the initial material shear modulus (calculated from the elastic modulus (EX) and Poisson's ratio (PRXY or NUXY)).

KEYOPT(5) provides various element printout options (see Element Solution). KEYOPT(6) provides the user with the option of selecting a reduced integration scheme for the volumetric terms of the strain energy function (available only with Mooney-Rivlin material).

A summary of the element input is given in "HYPER84 Input Summary". A general description of element input is given in Element Input. For axisymmetric applications see Axisymmetric Elements.

HYPER84 Output Data

The solution output associated with the element is in two forms:

  • Nodal displacements included in the overall nodal solution

  • Additional element output as shown in Table 84.1: "HYPER84 Element Output Definitions"

Several items are illustrated in Figure 84.2: "HYPER84 Stress Output". The element stress directions are parallel to the element coordinate system. A general description of solution output is given in Solution Output. See the ANSYS Basic Analysis Guide for ways to view results.

Figure 84.2  HYPER84 Stress Output

The Element Output Definitions table uses the following notation:

A colon (:) in the Name column indicates the item can be accessed by the Component Name method [ETABLE, ESOL]. The O column indicates the availability of the items in the file Jobname.OUT. The R column indicates the availability of the items in the results file.

In either the O or R columns, Y indicates that the item is always available, a number refers to a table footnote that describes when the item is conditionally available, and a - indicates that the item is not available.

HYPER84 Assumptions and Restrictions

  • The area of the element must be positive.

  • The element must lie in a global X-Y plane as shown in Figure 84.1: "HYPER84 Geometry" and the Y-axis must be the axis of symmetry for axisymmetric analyses. An axisymmetric structure should be modeled in the +X quadrants.

  • A face or edge with a removed midside node implies that the displacement varies linearly, rather than parabolically, along that face or edge. See Quadratic Elements (Midside Nodes) of the ANSYS Modeling and Meshing Guide for more information about the use of midside nodes.

  • The reduced integration of volumetric terms (KEYOPT(6) = 1) is computationally more expensive than the full integration, but may be necessary for problems where Poisson's ratio is greater than 0.48 and where significant hydrostatic stresses are likely to develop in any portion of the model. In these instances where reduced integration is used, the midside node form of the element (KEYOPT(1) = 1) is recommended.

  • If KEYOPT(1) = 0 is used with reduced integration, a finer mesh density should be used to maintain accuracy.

  • The reduced integration option is not used for the Blatz-Ko strain energy function (KEYOPT(2) = 1) or with the plane strain option of the 3-node triangular element.

  • Exceedingly large load steps will occasionally cause a zero energy mode (negative pivot message). Load step magnitudes should be reduced to eliminate this situation. In general, the more incompressible a material is, the smaller each load step should be to avert the zero energy modes. Alternatively, with Mooney-Rivlin materials, consider using HYPER74 which is less sensitive to numerical convergence problems.

  • The element may not be deactivated with the EKILL command.

HYPER84 Product Restrictions

There are no product-specific restrictions for this element.
