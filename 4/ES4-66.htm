4.66 FLUID66 Thermal-Fluid Pipe

FLUID66 is a three-dimensional element with the ability to conduct heat and transmit fluid between its two primary nodes and convect heat to two additional nodes (e.g., the pipe wall). Heat flow is due to the conduction within the fluid and the mass transport of the fluid. The convection may be related to the fluid flow rate. The element has two degrees of freedom, temperature and pressure, at each node (the pressure degree of freedom is inactive at the wall nodes).

The thermal-flow element may be used in a steady-state or transient thermal analysis. If the model containing the thermal-flow element is also to be analyzed structurally, the element should be replaced by an equivalent (or null) structural element. See Section 14.66 of the ANSYS Theory Reference for more details about this element.

4.66.1 Input Data

The geometry, node locations, and the coordinate system for this thermal-flow pipe element are shown in Figure 4.66-1. The element is defined by two primary nodes, two additional nodes if convection is desired, several real constants (see Table 4.66-1a), and the material properties. The length L of the element is determined from the two primary node locations. The fluid mass density (Mass/Length3) is input as property DENS. The convection film coefficient h (Heat/Length2*Time*Deg) is input as property HF. The thermal conductivity ks (Heat/Length*time*Deg) acts in the element longitudinal direction and is input as property KXX. The specific heat cp (Heat/Weight*Deg) is input as property C. The fluid viscosity is input as property VISC. In an axisymmetric analysis, such as for annular flow, the flow area, the convection areas, and all other input should be on a full 360° basis.

The coefficient of friction (input as property MU) is the starting value of the Moody friction factor (f). The friction factor for the first substep is assumed to be MU. If MU=0.0, the fluid flow capability is removed. If a friction factor table is not supplied, f is recomputed each substep after the first from smooth-pipe empirical correlations. The correlations are a function of Reynolds number (Re) and depend on whether the flow is laminar or turbulent (Re>2500). If the viscosity is zero, Re is not computed and f stays at its initial value. If a friction table is supplied (in the empirical data table), the friction factor is recomputed each substep from the table (using linear interpolation where necessary).

The empirical friction factor and film coefficient relationships are input through the empirical data table [TB]. If the table is not included, the friction factor is computed from the smooth pipe relationship and the film coefficient defined in the material property table is used. The empirical data table is accessed by the material number. The empirical data table is explained in detail in Table 4.66-1b.

The word PRES (or TEMP) should be input for the Lab variable on the D command and the pressure (or temperature) value input for the value. If a nodal heat (or fluid) flow rate is defined with the F command, input the word HEAT (or FLOW) for the Lab variable and input the flow rate for the value. Fluid weight effects are activated by specifying a nonzero acceleration vector [ACEL].

Element loads are described in Section 2.7. Element body loads may be input as heat generation rates at the nodes. The node J heat generation rate HG(J) defaults to the node I heat generation rate HG(I).

KEYOPT(6) is used for inputting flow losses (see Table 4.66-1a). Momentum losses in pipes due to bends, elbows, joints, valves, etc., may be represented by a fictitious (equivalent) length of pipe La. This equivalent length may be input directly or calculated from an input constant K, the hydraulic diameter D, and the friction factor f.

Use KEYOPT(9) to save velocities for use by the user-programmable feature USERCV (when accessed by SURF19 or SURF22) to determine velocity-dependent film coefficients.

A special option may be used to simplify the element if the fluid flow rate is known and only mass transport effects are desired. In this case, the flow rate does not have to be calculated from pressures, and the element can be used as a linear element. To invoke this option, input the known flow rate as real constant 9 (W) instead of with the F command. If the fluid flow rate is input as a real constant (W), the nodal fluid flow rate (if also input) is ignored.

4.66.3 Assumptions and Restrictions

The element must not have a zero length, so nodes I and J must not be coincident. Nodes K and L may be located anywhere in space, even coincident with I and J, respectively.

Compressibility and flow inertia effects of the fluid are not included in the element formulation. Viscous heating effects are also not included. The density and the area are assumed to remain constant for the element. The element uses a combined thermal conductivity and fluid conductance matrix. The resulting unsymmetric matrix requires twice as much memory storage for the solution as other ANSYS elements. (Unsymmetric matrices are always produced for this element.)

D must always be nonzero. A defaults to D2/4.0. MU and DENS must be nonzero if a flow solution is desired and the conductance coefficient (C) is not input. If C or W is input, MU, K, and La are not used. If the flow is specified at a node also having a specified pressure (p), the flow constraint is ignored. In general, flow is usually specified at the inlet, pressure at the outlet. HF must be nonzero for the four node element. The element is nonlinear and requires an iterative solution unless the flow rate, W, is defined by real constant 9. More substeps are required for convergence as the flow approaches zero. See the CNVTOL command for convergence control.

4.66.4 Product Restrictions

There are no product-specific restrictions for this element.
