HYPER86

3-D Hyperelastic Solid

MP ME ST <> <> <> <> <> <> PP ED

HYPER86 Element Description

HYPER86 is used for 3-D modeling of solid hyperelastic structures. This element is applicable to rubber-like materials with arbitrarily large displacements and strains. Both compressible and nearly incompressible materials may be modeled. The element is defined by eight nodes having three degrees of freedom at each node: translation in the nodal x, y, and z directions. The hyperelastic formulation is nonlinear and requires an iterative solution. Large deflections [NLGEOM] must be active in order to update the geometry in each substep.

Options allow you to select the type of strain energy density function, the element printout locations, and reduced integration. See HYPER86 in the ANSYS, Inc. Theory Reference for more details about this element. A similar element (without the hyperelastic formulation) is SOLID45. See also element HYPER58 for other hyperelastic formulations. A 2-D version of the hyperelastic element is described in HYPER84.

HYPER86 Input Data

The geometry, node locations, and the coordinate system for this element are shown in Figure 86.1: "HYPER86 Geometry". The element is defined by eight nodes and isotropic material properties. The element input data include the strain energy function (KEYOPT(2)) and the constants defining the Mooney-Rivlin strain energy function. The element coordinate system orientation is as described in Coordinate Systems.

Element loads are described in Node and Element Loads. Pressures may be input as surface loads on the element faces as shown by the circled numbers on Figure 86.1: "HYPER86 Geometry". Positive pressures act into the element. Temperatures may be input at the element nodes (for material property evaluation only). The node I temperature T(I) defaults to TUNIF. If all other temperatures are unspecified, they default to T(I). For any other input pattern, unspecified temperatures default to TUNIF.

KEYOPT(2) allows a choice of two strain energy functions. The Mooney-Rivlin function is valid for a wide range of nearly incompressible rubber and rubber-like materials. It is characterized by Poisson's ratio and constants a10 and a01 (entered with TBDATA or *MOONEY commands), which may be a function of temperature. See Mooney-Rivlin Hyperelastic Material Constants (TB,MOONEY) for the TBDATA command input. The Blatz-Ko function may be applicable to compressible foam-type polyurethane rubber materials. It is a function of the initial material shear modulus (calculated from the elastic modulus (EX) and Poisson's ratio (PRXY or NUXY)).

KEYOPT(5) provides various element printout options (see Element Solution). KEYOPT(6) provides the user with the option of selecting a reduced integration scheme for the volumetric terms of the strain energy function (available only with Mooney-Rivlin material).

A summary of the element input is given in "HYPER86 Input Summary". A general description of element input is given in Element Input.

HYPER86 Output Data

The solution output associated with the element is in two forms:

  • Nodal displacements included in the overall nodal solution

  • Additional element output as shown in Table 86.1: "HYPER86 Element Output Definitions"

Several items are illustrated in Figure 86.2: "HYPER86 Stress Output". The element stress directions are parallel to the element coordinate system. A general description of solution output is given in Solution Output. See the ANSYS Basic Analysis Guide for ways to view results.

The Element Output Definitions table uses the following notation:

A colon (:) in the Name column indicates the item can be accessed by the Component Name method [ETABLE, ESOL]. The O column indicates the availability of the items in the file Jobname.OUT. The R column indicates the availability of the items in the results file.

In either the O or R columns, Y indicates that the item is always available, a number refers to a table footnote that describes when the item is conditionally available, and a - indicates that the item is not available.

HYPER86 Assumptions and Restrictions

  • Zero volume elements are not allowed.

  • Elements may be numbered either as shown in Figure 86.1: "HYPER86 Geometry" or may have the planes IJKL and MNOP interchanged.

  • The element may not be twisted such that the element has two separate volumes. This occurs most frequently when the elements are not numbered properly.

  • All elements must have eight nodes.

  • A prism-shaped element may be formed by defining duplicate K and L and duplicate O and P node numbers (see Triangle, Prism and Tetrahedral Elements).

  • A tetrahedron shape is also available.

  • The reduced integration of volumetric terms (KEYOPT(6) = 1) is computationally more expensive than the full integration, but may be necessary for problems where Poisson's ratio is greater than 0.48 and where significant hydrostatic stresses are likely to develop in any portion of the model. This option is not used for the Blatz-Ko strain energy function (KEYOPT(2) = 1).

  • If the reduced integration option is used, a finer mesh density is recommended in those areas where higher hydrostatic stresses will occur.

  • Exceedingly large load steps will occasionally cause a zero energy mode (negative pivot message). Load step magnitudes should be reduced to eliminate this situation.

  • In general, the more incompressible a material is, the smaller each load step should be to avert the zero energy modes. Alternatively, with Mooney-Rivlin materials, consider using HYPER58 which is less sensitive to numerical convergence problems.

  • The element may not be deactivated with the EKILL command.

HYPER86 Product Restrictions

There are no product-specific restrictions for this element.
